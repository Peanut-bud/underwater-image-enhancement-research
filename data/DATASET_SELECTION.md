# 数据整理建议

## 1. 结论

基于当前 [数据](D:/科研/图像增强/数据) 目录的结构和抽样图像观察，建议按下面方式整理到工程标准入口 `data/`：

- `data/raw_field/`
  首选来源：
  - [03_第二次现场测试_20190113_20190128](D:/科研/图像增强/数据/03_第二次现场测试_20190113_20190128)
  - [04_第三次现场测试_20190519_20190525](D:/科研/图像增强/数据/04_第三次现场测试_20190519_20190525)

- `data/clean_source/`
  候选来源：
  - [1467-混凝土图像筛选](D:/科研/图像增强/数据/1467-混凝土图像筛选)

## 2. 判断依据

### 2.1 `03` 和 `04` 更适合 `raw_field`

这两批数据来自现场测试目录，本身就带有明显的真实退化特征：

- 水下浑浊
- 偏绿/偏黄
- 悬浮颗粒
- 局部过曝和照明反射
- 视野边缘暗角

这些特征和流程四、以及真实场景推理目标是对齐的，所以更适合归到 `raw_field`。

### 2.2 `1467-混凝土图像筛选` 只能作为“候选 clean source”

这批图像相比 `03` 和 `04`：

- 结构更稳定
- 混凝土表面纹理更清楚
- 目标区域更集中

所以它是目前最适合作为流程一 `J` 端候选的目录。

但必须强调：

**它仍然不是严格意义上的“无退化清晰图”。**

抽样图里依然能看到：

- 机器人灯光高光
- 边缘暗角
- 轻度色偏
- 水下成像痕迹

因此它更准确的定位应是：

- `候选清晰参考图`
- `相对更清楚的弱清晰图`
- `可用于阶段一起步的 clean_source`

而不是：

- 完美 Ground Truth
- 标准自然场景清晰图

## 3. 对流程一的影响

这意味着流程一现在可以往前跑，但要带着这个认知：

1. 你并不缺真实现场退化图。
2. 你当前缺的是“高质量 clean source”。
3. 现阶段最现实的方案，是先用 `1467-混凝土图像筛选` 作为阶段一的 `clean_source` 起步。
4. 后面如果能补充更高质量、退化更轻的混凝土/坝体图像，再替换或扩充 `clean_source` 会更合理。

## 4. 推荐整理策略

### 4.1 第一阶段

先不改动 [数据](D:/科研/图像增强/数据) 原始目录，只做工程内映射：

- 从 `03`、`04` 中筛选真实现场图片进入 `data/raw_field/images`
- 从 `1467-混凝土图像筛选` 中筛选相对清楚的图片进入 `data/clean_source/images`

### 4.2 第二阶段

等 `clean_source/images` 整理出第一批样本后，再执行：

1. [src/prepare_depth.py](D:/科研/图像增强/src/prepare_depth.py:1)
2. [src/prepare_synth.py](D:/科研/图像增强/src/prepare_synth.py:1)

### 4.3 第三阶段

如果后面发现 `1467` 这批图仍然太“脏”，就再引入新的更优 `clean_source`：

- 更清晰的混凝土表面图
- 更接近无浑浊条件的结构图
- 或人工进一步筛选 `1467` 中退化更轻的子集

## 5. 当前推荐结论

当前项目最合理的整理假设是：

- `raw_field`：来自 `03 + 04`
- `clean_source`：先来自 `1467-混凝土图像筛选`

这个方案不是最终最优，但它是目前最符合你现有资料结构、又能把流程一真正跑起来的方案。
