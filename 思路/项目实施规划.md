# 图像增强项目实施规划

## 1. 项目概述

本项目面向**人工光照水下场景图像增强**，目标是构建一个统一的研究工程，而不是简单调用若干现成算法做串联处理。

工程最终需要同时支持：

- 模型训练
- 单张图像推理
- 批量图像推理
- 定量评估
- 消融实验
- 结果可视化
- 论文图表输出

项目输出形式定义为：

```text
输入：单张水下原始图像
输出：增强后的图像
```

增强结果期望达到：

- 颜色更自然
- 亮度分布更均衡
- 暗区细节更可见
- 强反光区域更柔和
- 局部过曝过渡更自然
- 水底纹理与边缘更清晰
- 噪声与悬浮物干扰得到抑制

- 完全恢复真实纹理
- 精确重建原始细节

## 4. 总体技术路线

### 4.1 总体思路

不采用简单串联式增强流程：

```text
A算法 -> B算法 -> C算法 -> D算法
```

因为这种方式存在以下问题：

- 前一模块误差会向后传播并被放大
- 参数过多，不易统一调优
- 难以端到端训练
- 工程结构混乱
- 不利于论文消融实验

因此项目采用：

```text
单张输入图像
    ↓
基础预处理
    ↓
多分支退化感知
    ↓
主干增强网络
    ↓
特征融合
    ↓
图像重建输出
```

### 4.2 主干与辅助模块定位

模型主干固定为：

```text
频域增强 + 多支路差分卷积细节恢复
```

辅助模块固定为：

```text
高光感知 + 非均匀光照校正 + 混合曝光校正 + 注意力融合 + 噪声抑制
```

整体定位为：

- 主干负责：偏色、浑浊、模糊、纹理弱
- 辅助分支负责：反光、过曝、暗区、曝光不均、局部退化

## 8. 模型总体结构设计

建议模型结构为：

```text
Input
 ↓
Shallow Feature Extraction
 ↓
Highlight-Aware Branch
Illumination Correction Branch
Mixed Exposure Branch
Frequency-domain Enhancement Branch
Differential Convolution Detail Branch
Noise Suppression Branch
 ↓
Attention Fusion
 ↓
Reconstruction Head
 ↓
Enhanced Output
```

关键原则：

- 逻辑上是多分支协同
- 不是完全串行流水线
- 所有模块应支持单独开关
- 所有模块应支持消融实验

## 9. 模型模块详细规划

### 9.1 浅层特征提取模块

作用：

- 提取基础颜色、纹理、边缘信息
- 作为所有分支的共享输入特征

要求：

- 轻量
- 保持空间分辨率
- 易于接多分支结构

### 9.2 高光感知分支

目标：

- 检测强反光、局部过曝、高亮异常区域

作用：

- 输出高光相关的区域先验
- 为后续曝光修正和融合提供引导

输出建议：

- highlight mask
- 高亮注意力图
- 区域置信权重图

### 9.3 非均匀光照校正分支

目标：

- 解决左亮右暗、局部照明不均、暗区细节不足

作用：

- 估计照明分布
- 平衡亮区与暗区
- 减少照明不均带来的增强偏差

输出建议：

- illumination map
- 光照校正特征图

### 9.4 混合曝光校正分支

目标：

- 同时处理局部过亮与局部过暗共存问题

作用：

- 对不同曝光区域进行区域级协调
- 改善曝光一致性

输出建议：

- exposure feature
- 区域级曝光补偿特征

### 9.5 频域颜色与细节增强模块

定位：

- 项目主干创新之一

目标：

- 校正偏色
- 提升低频颜色自然性
- 恢复高频纹理细节
- 改善浑浊和低对比度

实现方向：

- FFT 频域增强
- 小波分解增强
- 双域联合增强

第一版建议：

- 先实现一个稳定的频域增强块
- 保持接口清晰，后续再扩展

### 9.6 多支路差分卷积细节恢复模块

定位：

- 项目主干创新之一

目标：

- 恢复纹理
- 恢复边缘
- 强化局部结构响应

作用：

- 用差分卷积捕获高频变化
- 从多方向、多尺度恢复纹理信息

要求：

- 不过锐化
- 不把噪声误增强为纹理
- 和噪声抑制分支配合工作

### 9.7 噪声抑制模块

目标：

- 抑制颗粒噪声与悬浮物干扰
- 限制锐化或频域增强带来的噪声放大

作用：

- 平衡细节恢复与噪声抑制
- 让结果更干净而不是更碎

输出建议：

- denoise feature
- 噪声置信图
- 特征门控权重

### 9.8 注意力融合模块

目标：

- 对不同分支输出进行加权融合

作用：

- 自动决定哪些区域更依赖颜色恢复
- 哪些区域更依赖曝光修正
- 哪些区域更依赖细节增强
- 哪些区域更需要抑制噪声

推荐能力：

- 通道注意力
- 像素注意力
- 多尺度融合

### 9.9 图像重建头

目标：

- 从融合特征中恢复最终 RGB 图像

要求：

- 输出分辨率与输入一致
- 边缘过渡自然
- 避免棋盘格与重建伪影

## 10. 损失函数设计

### 10.1 第一阶段损失组合

第一版先采用稳定损失组合：

- `L1 Loss`
- `SSIM Loss`
- `Edge Loss`

目的：

- 保证基本像素恢复
- 保证结构相似性
- 提升边缘清晰度

### 10.2 第二阶段损失组合

在第一版稳定后加入：

- `Color Loss`
- `Perceptual Loss`

目的：

- 提升颜色自然度
- 提升主观视觉质量

### 10.3 第三阶段扩展损失

后续可加入：

- `Highlight Region Loss`
- `Exposure Consistency Loss`
- `Contrastive Loss`

目的：

- 针对高光区域做约束
- 保证区域间曝光一致
- 提升特征层面的清晰图接近度

### 10.4 损失引入顺序

固定顺序：

1. `L1 + SSIM + Edge`
2. 加 `Color`
3. 加 `Perceptual`
4. 再加区域损失和对比损失

这样便于：

- 排查训练不稳定原因
- 逐项做消融实验
- 写论文时清楚说明各损失贡献
