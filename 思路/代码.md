你现在最需要的是让工程继续朝“单图输入 -> 增强图输出”的完整模型推进，所以建议后续优先按这个顺序参考：

```text

2. HDR CNN
   用途：处理高光区域细节丢失 / 饱和亮区

3. Over-exposure Correction
   用途：处理严重过曝区域

4. FUnIE-GAN
   用途：水下图像增强基线

5. U-shape Transformer
   用途：深度学习水下增强强基线

6. Water-Net
   用途：学习多输入融合结构

7. Ucolor
   用途：学习多颜色空间颜色校正

   用途：自己复现频域增强 + 差分卷积主干
```

- 先看有明确代码、而且和你当前问题最接近的仓库。

# 八、后面继续写代码时的参考规则

这一节是给后面工程实现时直接看的。

## 1. 不把外部论文仓库整仓照搬进主工程

原则：

- 外部仓库用于读结构、读模块、读输入输出接口。
- 主工程仍然保持我们自己的 `src/models`、`src/losses`、`src/train.py`、`src/infer.py` 体系。

## 2. 先参考模块思想，再决定是否复现细节

例如：

- 频域增强：先参考黄奕程等思路，再用自己的工程方式实现。
- 高光分支：先参考 HDR CNN 和高光去除方向，再设计适合自己场景的轻量分支。
- 非均匀光照：先参考 UNIR-Net，再决定是做 illumination map 还是 feature modulation。

## 3. 主干优先自己实现，辅助分支优先借鉴公开代码

当前推荐策略：

- 主干：
  - 频域增强
  - 差分卷积细节增强
  - 由我们自己维护实现

- 辅助分支：
  - 高光
  - 曝光
  - 光照
  - 可以更多参考已有公开代码

## 4. 先做“能接进去”的轻量版，不一开始追求完整复现

例如：

- 高光分支先做 `highlight_map`
- 非均匀光照分支先做 `illumination_map`
- 曝光分支先做区域级 feature correction

不要一开始就把整篇论文复杂结构完整搬进来。

## 5. 参考优先级固定

后续加模块时建议按下面这个优先级：

# 九、当前工程 Python 文件与论文代码的对应关系

这一节直接按我们工程里的实际 Python 文件来写。

后面继续开发时，可以先看“你现在要改哪个文件”，再看它后面对应参考哪篇论文或哪个仓库。

## 1. 入口文件

### `src/infer.py`

主要职责：

- 单图或批量推理入口
- 读取配置
- 加载模型
- 调用推理流程
- 保存增强结果

优先参考：

- `FUnIE-GAN`
- `U-shape Transformer`
- `Water-Net`
- `Ucolor`

为什么：

- 这类仓库通常都比较清楚地给出了推理入口、权重加载方式、输入输出组织方式。
- 它们最适合参考“单图输入 -> 增强图输出”的工程入口设计。

后续如果这里继续扩展，重点看：

- 不同模型的权重切换方式
- 单图/批量推理参数组织
- 输出命名和可视化对比图组织

### `src/train.py`

主要职责：

- 训练总入口
- 读取训练配置
- 构建 dataset / dataloader / model / loss / trainer

优先参考：

- `FUnIE-GAN`
- `U-shape Transformer`
- `Ucolor`
- 王悦等论文

为什么：

- 这些更适合参考训练入口结构、训练参数组织、验证流程和 loss 调用方式。
- 王悦等论文后面主要参考训练策略和附加损失，不是参考入口脚本本身。

### `src/eval.py`

主要职责：

- 后续独立评估入口
- 统一组织指标计算和结果导出

优先参考：

- `FUnIE-GAN`
- `U-shape Transformer`
- `Ucolor`

为什么：

- 这些基线仓库更适合参考评估流程组织。

## 2. 数据与预处理文件

### `src/models/minimal_enhancer.py`

主要职责：
